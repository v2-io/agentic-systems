### 1. R2 naming aggregator + score-card + patterns view (LANDED)

Built `bin/naming-r2-aggregate.rb` (~1500 lines) with three outputs:

- **Score-card** at `msc/naming/r2-aggregate-table.md` — single unified table sorted by `max(score/n)` descending. Markers: `▸` for current corpus name, `⊕` for any add-alias votes; **bold** for leader.

- **Patterns view** at `msc/naming/r2-patterns.md` — cross-cutting categorical groupings (defended keeps / rename signals / add-alias landings / contested / negative) + cross-cutting patterns (Greek-vocabulary cluster, math-symbol add-alias pattern, Class-N taxonomy, Pearl hierarchy) + caveats + suggested reading order. Generated for the downstream renaming agent that will produce first-pass landings.

Substance factor through ~6 refinement iterations with Joseph in real time:

```
relative_length = note_chars / bullet_chars
effort          = min(1, relative_length)
factor          = (0.7 + 0.3 × effort) × (1.0 + novelty)
factor         *= 1.2 if vote.top_pick
factor         *= 1.2 if vote.category == 'canonicalize'
```

Top-pick × 1.2 backed by a benchmark showing it acts as tiebreaker in 20/20 multi-+2-per-target cases (otherwise mostly redundant with +2). Canonicalize × 1.2 captures excavated-from-prose evidence per the principles file. Filters: ≥2 R2 voters AND alternatives proposed (drops 16 uncontested keeps like `action fluency`, `adaptive system` where no alternative was ever offered at any phase).

The R1 cohort (10 voters) is folded in as one synthetic voter on the R2 scale — preserves R1's collective verdict without re-counting per-agent. Detailed rationale in the script header.

Cohort coverage: 122 multi-R2-voter targets out of 629; ~40% of all targets have a metadata anchor in the naming-context-map (139 by master-list `segment_link` + 110 by grep first-mention); the remaining ~60% are where Phase 2 semantic anchoring becomes load-bearing.
