## Principles-Observations

### Cold-start instruction effectiveness

The cold-start rule (do not read prior vote files, aggregations, or brainstorms) worked well. My votes converge independently on several Round-1 consensus renames (additive-coordinate-forcing → forced-coordinates; separability-pattern → separability-ladder) while diverging on others (I am weaker on certain proposed changes that Round-1 may have favored). This suggests the cold-start is effective at maintaining diversity.

### Architectural invariants understood

The role-prefix discipline and subject-noun-first principle are clear from FORMAT.md and CLAUDE.md. I voted only on subject-nouns and deferred all role-prefix considerations. The principle *"name the thing defined, not the role the segment plays"* is well-implemented in most segments (e.g., #result-persistence-condition, #form-strategy-complexity-cost); a few show the anti-pattern (e.g., #deriv-causal-ib-exploration names the effect rather than the engine — this is my +2 rename candidate).

### Greek vocabulary alignment

The project's coherent Greek-rooted vocabulary (chronica, prolepsis, aisthesis, aporia, epistrophe, praxis, logogenic, logozoetic) is elegant and justified. I voted to canonicalize the five cycle phases with paired English glosses rather than renaming them — the Greek carries semantic distinctions English flattens (e.g., "perplexity" for aporia preserves the *productive* sense that "prediction error" loses). No conflicts with this aesthetic register observed.

### Evaluation criteria applied

I explicitly used the communal-imagination test, renamed-from-now-sounds-weird test, and scope-honesty criterion for major votes. The satisfaction-gap / control-regret pair is the canonical example of names doing load-bearing work — changing either would displace vast prose conventions. The persistence-envelope and observability-dead-zone candidates both pass the communal-imagination test in my judgment.

### Unclear criteria and gaps

1. **Scope-honesty for meta-segments is under-specified in practice.** The principles file notes the special case (meta-segment names must be broad enough to cover all instances) but I had to interpret this myself while voting on candidates like "observability dead zone" — is it meta enough? The conflict between narrowness (more memorable) and breadth (covering all uses) is real and I'm uncertain how to weight it.

2. **Plural vs. singular for collection nouns.** "Separability ladder" vs "separability ladders" — I voted for singular (pattern was singular; ladders are a single system). No explicit guidance on this in the principles file.

3. **When to add aliases vs. when to rename.** I voted to add English aliases for α, R, and η* (symbols that are standard but could be read aloud more naturally). The principles file does not clearly distinguish when an add-alias vote is preferable to a rename of the symbol itself. This may be correct — symbols are stable and shouldn't be renamed — but the guidance could be clearer.

4. **Exhaustiveness threshold for name-unnamed.** The principles file gives the rule-of-thumb *"if you would use the name repeatedly in a conversation about the theory, it's eligible."* I found ~13 unnamed candidates; am uncertain whether I've been thorough or excessive. Some (e.g., "deliberation threshold") feel borderline — they are discussed in segments but the *noun* is not standard enough to be used in quick conversation yet. Others (e.g., "observability dead zone") feel ripe for promotion.

### Strongest votes

1. **#def-satisfaction-gap, #def-control-regret (both +3 keep).** These two names do load-bearing work for the diagnostic core. The 2×2 table (world permits / you achieve) makes intuitive sense *because the names carry the axes*. Changing either would have cascading prose costs.

2. **#result-persistence-condition (+3 keep).** This name is canonical across five distinct domains (control, RL, organizations, software, agencies). It appears so frequently and so centrally that rename would be impossibly costly. The name is excellent and cannot be improved.

3. **#disc-additive-coordinate-forcing → #disc-forced-coordinates (+3 rename).** The current name undersells the Čencov 4th instance (which is not Cauchy-FE). "Forced coordinates" is more evocative, shorter, and more memorable. Passes the renamed-from-now-sounds-weird test cleanly (old name sounds narrow in hindsight).

4. **#disc-separability-pattern → #disc-separability-ladder (+3 rename).** The "rung structure" across the seven difficulty axes is isomorphic to a ladder. "Ladder" is more evocative than "pattern" and would age better in conversation.

5. **#deriv-causal-ib-survival (+2 rename, from #deriv-causal-ib-exploration).** The foundational thing being derived is a *survival imperative*; exploration is the downstream behavioral consequence. Naming the cause rather than the effect aligns with the subject-noun-first principle and prevents misreading the derivation as being about exploration pedagogy.
