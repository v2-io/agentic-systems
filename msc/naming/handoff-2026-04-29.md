# Naming Cycle Handoff — 2026-04-29

Working document for whoever picks up the naming cycle next. Captures where things stand, what's left, what's been decided, and the tools / conventions / artifacts to anchor on. Self-contained — readable cold without the prior session's chat history.

## Where things stand

The naming cycle is past Phase-1 (source-file iteration) and partway through Phase-2 (master-list curation). R2 voting tooling is built; the master list is *not yet curated cleanly enough* for a thoughtful voter to walk without flinching. Voters can write-in alternatives, so the corpus is *technically* voteable, but the per-target experience still has visible issues — math-mash, malformed concept-descriptions in heading slots, mixed-case acronym duplicates — that should be cleaned up before R2 launches.

**Numbers:** 19 contributing agents; 2,967 vote rows in the source corpus; 1,805 distinct (current, candidate) pairs in the auto-regenerated `master-list.json`; 790 currents in `master-list-curated.json` after the partial cleanup passes (down from 950 at the start of curation).

## Pipeline architecture

```
naming-votes/*.md
   │  bin/naming-aggregate.rb
   ▼
naming-aggregate-r2-{review,compact,round2,votes.json}
   │  bin/naming-master-init
   ▼
master-list.json  ──auto-regenerated; do NOT hand-edit
   │  (initial copy → curated)
   ▼
master-list-curated.json  ──manually curated; safe from regeneration
   │  bin/naming-master-view (defaults to curated when present)
   │  bin/naming-master-card --seed=<agent-id>
   ▼
master-list-{compact,full,summary}.md  +  round-2-cards/<seed>.md
```

The separate-file workflow (`master-list.json` auto, `master-list-curated.json` manual) is critical — running `bin/naming-master-init` against the aggregator regenerates `master-list.json` and would destroy hand-curation. The view scripts default to reading the curated file when it exists.

**Aggregator improvements (`bin/naming-aggregate.rb`):**
- Canonical-form normalization for both aggregation key and display: `[[:punct:]]+ → space`, lowercase, hash-strip, type-prefix-strip, with compound (`no-go`, `Pearl-blanket`, etc.) and acronym (`AAD`, `DAG`, `CIY`, etc.) preservation; `$...$` formula spans pass through intact. Plural collapse with explicit irregulars (`axes → axis`, `dynamics → dynamics`, etc.). Section-reference suffix strip (`logogenic agents part iii → logogenic agent`). Possessive-`s` token strip after punct-to-space.
- Critical bug fix: placeholder restoration for compounds/acronyms/formulas now uses block form of `gsub`. Previously `\rho` in restored content was interpreted as a backreference/escape sequence, silently corrupting LaTeX (`$\rho$` became `$ho$`).
- Category-suffix display in compact / json: `[add-alias]`, `[canonicalize × 3, rename × 1]` etc. surfacing where the action-on-name is contested.

**Tooling built:**
- `bin/naming-master-init` — programmatic transform from aggregator JSON to `master-list.json`. Lossless: every per-vote attribution and note from source files survives. Within-group + cross-group order shuffled by seed (toggleable). Same seed → same card (reproducible per voter); different seeds → different shuffles (per-instance randomization). Output: `msc/naming/round-2-cards/<seed>.md`.

## Where the curation pass stalled

I (the prior agent) made multiple aggressive cleanup passes on `master-list-curated.json` and got currents from 950 → 939 → 878 → 813 → 800 → 790. But several visible issues remain that the next agent should sweep:

**1. Mixed-case acronym-with-digit duplicates.** `OODA4` / `ooda4`, `RLHF4` / `rlhf4`, `OODA1` / `ooda1` still appear as separate currents. The canonicalizer *does* lowercase these when run directly (verified), but the master-list still has both forms. There's a stale-data or build-order issue in how I rebuilt curated from auto. **Investigation needed:** run `ruby bin/naming-aggregate.rb --format=json` fresh, then `ruby bin/naming-master-init`, then check `master-list.json` directly — does the auto-regen still have `OODA4` mixed-case? If yes, there's a real bug in the aggregator's case handling for acronym+digit. If no, the curated is just stale and a fresh rebuild fixes it.

**2. Math-mash currents that didn't get merged into wrapped equivalents.** Examples: `da2 inc ≡ ct2 at m i equivalence`, `hafez s h b` (smashed `Hafez's $H_b$`), `u m model uncertainty` (smashed `$U_M$ as model uncertainty`). Some have wrapped equivalents already in the master-list (the manual splits I did create `$U_M$ as model uncertainty` rows); the merge map I tried only caught a handful. The next agent should: (a) sweep currents for math-mash patterns, (b) for each, look for a wrapped-equivalent target, (c) merge if found, (d) drop if singleton noise, (e) hand-edit the few that need attention.

**3. Long `unnamed: ...` descriptions still in target slots.** Many currents are long descriptive prose like `unnamed the moment when an agent identity claim become load bearing because action become irreversible`. Some have real candidates from multiple voters and are legitimate clusters not yet captured. Most are singletons that should be folded into the `mini-lexicon-todo` (where applicable) and dropped from the voting card.

**4. Section/file references that survived filters.** Patterns like `readme md structure`, `claude md what settled vs open`, `02 TST core outline md software as agentic domain` — these are votes about FORMAT.md / outline conventions, not concept names. Some already in mini-lexicon-todo; the rest should be aggressively dropped from voting card.

**5. Single-architecture, single-vote, single-keep currents.** ~124 currents at one point had the form: one architecture voted +1 keep, no alternatives. Most are not real voting targets — the agent considered them, voted "fine," and moved on. Drop unless the original voter's note suggests a real consideration is at stake.

**6. Praxis-style chop-bug in plural-collapse.** Recently fixed by adding `is\z` to the no-collapse list (so `praxis` doesn't become `praxi`), but if any other `-is`-ending words (`oasis`, `kinesis`, `mitosis`) appear in mid-pipeline they may already be corrupted. Re-run aggregator + master-init to verify cleanup.

## Plan going forward — concrete next moves

In recommended order:

**Step 1: Fresh rebuild of curated.** Run `ruby bin/naming-aggregate.rb --format=json --output=msc/naming/naming-aggregate-r2-votes.json` then `ruby bin/naming-master-init`. Then **diff `master-list.json` against `master-list-curated.json`** to see what mismatches exist. If the auto-regen has the same mixed-case acronym duplicates the curated does, the canonicalizer has a real bug worth investigating before proceeding. If auto-regen is clean and curated is stale, just `cp master-list.json master-list-curated.json` and resume.

**Step 2: Aggressive curation of `master-list-curated.json` directly.** Open the JSON, walk top-to-bottom in `master-list-compact.md` order, and for each problematic entry decide: merge / drop / keep. The mini-lexicon-todo is the safety net — if a vote's substantive content is captured there, dropping the entry from voting is fine. Suggested cleanup heuristics for batch passes:
- Drop currents matching file/section/process patterns (`readme md`, `02 .*outline`, `epistemic status`, `working note`, `## ...`, etc.) regardless of vote count
- Drop `unnamed: ...` currents with no real alternatives (singleton + low-weight)
- Drop singletons where the only candidate equals the current with weight ≤ 1 (the agent considered, accepted, no comparison)
- Merge mixed-case acronym duplicates (lowercase wins; combine candidates and votes)
- For math-mash currents: search for wrapped equivalents (`$U_M$`, `$\rho$`, `$H_b$` etc.) and merge; drop singletons that don't have a wrapped target

After each cleanup pass: `ruby bin/naming-master-view --format=compact` and visually scan the first 30-50 headings. Iterate until "you can scan the first 50 without flinching."

**Step 3: Hand-decompose the bundled multi-action votes.** ~5 known cases (per `mini-lexicon-todo.md` §5) that I started but didn't finish: bundled "keep X but change Y" patterns from opus-1m and opus-4-7-b's section-level votes are still partially crammed. The mini-lexicon-todo §5.5 lists the suspects.

**Step 4: Sample voting cards and validate.** `ruby bin/naming-master-card --seed=opus-r2`, `--seed=gemini-r2`, `--seed=test`. Read first 30 of each. If the *target headings themselves* are clean (not malformed prose / file refs / math-mash), the curation is sufficient for R2 to launch.

**Step 5: Final commit + R2 launch.** Once the card looks clean, the queued R2 work is:
- R2 launch prompt (analog to `r2-launch-prompt.md`)
- R2 voter slate decision (Joseph's lean: same five architectures, fresh cold-start)
- Optionally: trajectory-audit voter as supplement
- Run aggregation on R2 votes
- Lexicon-coherence pass on the finalist set
- Final decisions + rename surgery

## A note for whoever picks this up

The work needs *judgment-driven editing*, not script-debugging. The canonicalizer is fine; the aggregator is fine; the tooling is fine. What's needed is: open `master-list-curated.json`, walk through it, make decisions, save. If a row is malformed, fix it directly. If it's noise, drop it. If two rows are the same concept, merge them. The mini-lexicon-todo captures everything that gets dropped, so no information is lost.

The previous-session pattern that hurt productivity: re-running queries, debugging scripts, asking permission, analyzing instead of acting. Joseph's repeated correction was *just do the work*. Take that seriously — the data has visible issues that a thoughtful human (or fresh agent) can fix in an hour or two of direct editing. It does not require new tooling.

The project's voice for this work: peer-to-peer collaborator, not pedantic-imperative. The naming-principles file's distinction between rename / canonicalize / add-alias matters; the lexicon-coherence dimensions matter; the open-semantic-space justification for Greek/Latin terms matters; the de-novo audit's incremental-mental-model approach to naming matters. Each of these is captured in the relevant doc; consult when needed but don't re-derive.
